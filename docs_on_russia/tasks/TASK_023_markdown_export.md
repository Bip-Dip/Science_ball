## Цель

Экспортировать обоснованные ответы и доказательства в Markdown.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-022
- SDD требования к экспорту ответов/демо

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/api/routes/exports.py`
- `backend/app/api/router.py`
- `backend/app/services/exports/markdown_export_service.py`
- `backend/app/schemas/exports.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Создать Markdown экспорт для ответа, источников, confidence, противоречий, пробелов.
- Включить ссылки на исходный документ/чанк.
- Опционально сохранять артефакт экспорта в MinIO, если существующие абстракции хранилищ это поддерживают.
- Добавить frontend кнопку/поток скачивания.

---

## Явно не делать

Не делать:

- экспортировать неподтверждённые/выдуманные утверждения
- включать скрытые документы
- хранить секреты
- реализовывать PDF экспорт, если это не тривиально и явно не входит в объём
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
