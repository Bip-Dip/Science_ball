# TASK_019_fact_review_api

## Цель

Реализовать API проверки фактов для человеческой верификации, отклонения, комментариев и истории версий.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-018
- SDD ручная верификация, версионирование, требования аудита

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/api/routes/facts.py`
- `backend/app/api/router.py`
- `backend/app/schemas/facts.py`
- `backend/app/services/review/fact_review_service.py`
- `backend/app/repositories/facts.py`
- `backend/app/repositories/audit_logs.py`
- `backend/app/models/fact.py`
- `backend/app/models/audit_log.py`
- `backend/tests/unit/test_fact_review_service.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Список ожидающих фактов с фильтрами.
- Подтверждение/отклонение фактов с комментарием проверяющего.
- Создание записей журнала аудита для действий проверки.
- Сохранение истории версий при редактировании значений факта.
- Уважение access_level и будущих RBAC границ.

---

## Явно не делать

Не делать:

- реализовывать полную аутентификацию, если её ещё нет
- удалять исходные доказательства
- разрешать нетрассируемые изменения фактов
- реализовывать frontend
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
