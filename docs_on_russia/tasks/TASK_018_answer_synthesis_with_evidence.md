# TASK_018_answer_synthesis_with_evidence

## Цель

Генерировать обоснованные ответы из извлечённых доказательств с использованием LLMGateway.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-017
- SDD требования к ответам и принцип "LLM не источник истины"

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/api/routes/answers.py`
- `backend/app/api/router.py`
- `backend/app/services/answers/__init__.py`
- `backend/app/services/answers/answer_service.py`
- `backend/app/services/answers/prompt_builder.py`
- `backend/app/schemas/answers.py`
- `backend/tests/unit/test_answer_service.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Предоставить `POST /api/v1/answers` или эквивалентный endpoint ответа.
- Получать/использовать доказательства из поискового сервиса.
- Промптить LLM только с доказательствами и явным правилом запрета выдумывания.
- Возвращать краткое содержание, источники, confidence, противоречия, пробелы в знаниях, где возможно.
- Если доказательств нет, возвращать ответ об отсутствии ответа вместо галлюцинаций.

---

## Явно не делать

Не делать:

- позволять LLM выдумывать неподтверждённые факты
- обходить поисковые/доступные фильтры
- раскрывать секреты
- вызывать провайдера напрямую
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
