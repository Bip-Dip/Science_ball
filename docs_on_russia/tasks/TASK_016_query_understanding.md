# TASK_016_query_understanding

## Цель

Реализовать понимание запроса, которое преобразует вопросы пользователя в валидированный поисковый intent.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-015
- SDD гибридные поисковые фильтры, запросы на русском/английском

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/services/query/__init__.py`
- `backend/app/services/query/query_understanding.py`
- `backend/app/schemas/query.py`
- `backend/tests/unit/test_query_understanding.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Возвращать валидированный Pydantic поисковый intent с текстом запроса, фильтрами, числовыми ограничениями, языковыми подсказками.
- Использовать детерминированный fallback, когда LLM отключена или падает.
- Использовать только LLMGateway, никогда напрямую провайдер.
- Валидировать и очищать LLM вывод перед использованием.
- Уважать access_level как обязательный вход от вызывающего/сессионного контекста.

---

## Явно не делать

Не делать:

- выполнять поиск Elasticsearch
- генерировать финальные ответы
- обходить LLMGateway
- доверять сырому LLM JSON без валидации
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
